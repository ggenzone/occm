バックアップファイルからのボリュームデータの復元

2020年12月4日寄稿者 このページのPDFをダウンロード

バックアップはクラウドアカウントのオブジェクトストアに保存されるため、特定の時点からデータを復元できます。保存したバックアップファイルからボリューム全体を復元できます。または、少数のファイルのみを復元する必要がある場合は、保存したバックアップファイルから最大8つの個別のファイルを（一度に）復元できます。

ボリューム全体の復元は、バックアップファイルの作成に使用したBackup toCloudサービスの一部です。作業環境でBackupto Cloudサービスをアクティブ化すると、同じ作業環境でボリューム全体を復元したり、同じクラウドアカウントを使用している別の作業環境に復元したりできます。バックアップからのボリュームの復元を参照してください。

バックアップファイルから個々のファイルを復元するには、実際には2つのサービスが使用されます。 Backup to Cloudサービスを使用して、上記のようにバックアップファイルを作成します。ただし、単一ファイルの復元を開始すると、復元サービスを使用してファイルが復元されます。ここでも、同じクラウドアカウントを使用している、同じ作業環境のボリューム、または別の作業環境のボリュームにファイルを復元できます。バックアップからのファイルの復元を参照してください。

オンプレミスのONTAPシステムから作成されたバックアップは、バックアップが存在する場所と同じクラウドプロバイダーを使用するCloud VolumesONTAPシステムにのみ復元できます。

バックアップからのボリュームの復元

バックアップファイルからボリュームを復元すると、CloudManagerはバックアップのデータを使用して新しいボリュームを作成します。同じ作業環境内のボリューム、またはソース作業環境と同じクラウドアカウントにある別の作業環境にデータを復元できます。

ステップ

  1. 作業環境を選択してください。

  2. クリック [バックアップの表示]を選択します。

  3. 復元するボリュームの行を選択し、 [バックアップリストの表示]をクリックします。

  4. 復元するバックアップを見つけて、[復元]アイコンをクリックします。

  5. [バックアップを新しいボリュームに復元]ページに入力します。

    1. 作業環境を選択してください。

    2. ボリュームの名前を入力します。

    3. [復元]をクリックします。

結果

Cloud Managerは、選択したバックアップに基づいて新しいボリュームを作成し、その作業環境に移動して、復元の進行状況を確認できるようにします。この新しいボリュームは、必要に応じて管理できます。

バックアップからのファイルの復元

ボリュームのいくつかのファイルのみを復元する必要がある場合は、ボリューム全体を復元するのではなく、個々のファイルを復元することを選択できます。バックアップファイルから個々のファイルを復元するには、復元サービスを使用します。同じクラウドアカウントを使用している同じ作業環境のボリューム、または別の作業環境にファイルを復元できます。

バックアップファイルのボリュームから一度に最大8つのファイルを復元できます。 8つを超えるファイルを復元する必要がある場合は、復元プロセスをもう一度実行できます。すべてのファイルは、選択した同じ宛先ボリュームに復元されます。

現時点では、単一ファイルの復元はAWSでのみサポートされています。

プロセスは次のようになります。

  1. [サービス]パネルから復元サービスをアクティブにします。

    復元インスタンスはクラウドプロバイダーの環境にデプロイされ、アクティブな復元ジョブがない間はコストを節約するためにシャットダウンされます。

  2. ボリュームから1つ以上のファイルを復元する必要がある場合は、[復元]タブをクリックして、ファイルが存在するバックアップファイル（スナップショット）を選択します。

    Restoreインスタンスが起動し、バックアップファイルに存在するフォルダーとファイルが表示されます。

  3. そのバックアップからファイル（または最大8つのファイル）を選択し、ファイルを書き込む場所（作業環境、ボリューム、およびフォルダー）を選択して、[復元]をクリックします。

    Restoreインスタンスはファイルを復元し、1時間操作がないとインスタンスがシャットダウンされます。

詳細

費用

Backup to CloudサービスとRestoreサービスの両方のコストについては、このトピックを参照してください。

AWSの権限が必要です

Cloud Managerにアクセス許可を提供するIAMロールには、 AWS要件で説明されているように、最新のCloudManagerポリシーからのS3アクセス許可が含まれている必要があります。

さらに、単一ファイルの復元のポリシーには、次の権限が必要です。

 "Action": [ "ec2:DescribeInstanceTypeOfferings", "ec2:startInstances", "ec2:stopInstances", "ec2:terminateInstances" ],
アクセス

Restoreインスタンスには、パブリッククラウド環境内のリソースとプロセスを管理するためのアウトバウンドインターネットアクセスが必要です。

復元サービスのアクティブ化

これは、復元インスタンスを作成するための作業環境ごとの1回限りの操作です。

この作業環境では、クラウドへのバックアップサービスがすでにアクティブ化されている必要があります。
ステップ

  1. 作業環境を選択してください。

  2. 復元サービスから[アクティブ化]ボタンをクリックします。

復元インスタンスはクラウドプロバイダーの環境にデプロイされてからシャットダウンされます。インスタンスがアクティブな場合にのみ復元コストが発生するように、コストを節約するためにインスタンスが停止されます。

バックアップファイルからの単一ファイルの復元

次の手順に従って、ボリュームバックアップから新しいボリュームに最大8つのファイルを復元します。この機能はライブブラウジングを使用するため、バックアップファイル内のディレクトリとファイルの現在のリストを表示できます。

バックアップファイルはNetAppスナップショットテクノロジを使用して作成されるため、UIの表現では各バックアップファイルが「スナップショット」と呼ばれることに注意してください。

次のビデオは、単一のファイルを復元するための簡単なウォークスルーを示しています。

ONTAPのバージョンは、ソースおよび宛先のCloud VolumesONTAPシステムで9.6以降である必要があります。
ステップ

  1. [復元]タブをクリックします。

    復元ダッシュボードが表示されます。

  2. [ファイルの復元]ボタンをクリックします。

  3. [ソースの選択]ページで、作業環境、ボリューム、およびファイルを復元する日付/タイムスタンプのあるスナップショットを選択します。

  4. [続行]をクリックすると、インスタンスの復元が開始されます。数分後、Restoreインスタンスはボリュームスナップショットからのフォルダーとファイルのリストを表示します。

  5. [ファイルの選択]ページで、復元する1つまたは複数のファイルを選択し、[続行]をクリックします。

    • 検索アイコンをクリックしてファイルの名前を入力すると、ファイルに直接移動できます。

    • 表示されている場合は、ファイル名をクリックできます。

    • を使用してフォルダ内のレベルを下に移動できますファイルを見つけるために行の終わりにあるボタン。

      ファイルを選択すると、ページの左側に追加されるため、選択済みのファイルを確認できます。必要に応じて、ファイル名の横にあるxをクリックして、このリストからファイルを削除できます。

  6. [宛先の選択]ページで、ファイルを復元する作業環境、ボリューム、およびフォルダーを選択します。

    ファイルを復元するときの場所にはいくつかのオプションがあります。

    上記のように、 [ターゲットフォルダの選択]を選択した場合：

    • 任意のフォルダを選択できます。

    • フォルダにカーソルを合わせてクリックできます行の最後でサブフォルダーにドリルダウンしてから、フォルダーを選択します。

    ソースファイルと同じ作業環境とボリュームを選択した場合（アイコン）、[ソースフォルダーパスの維持]を選択して、ファイルまたはすべてのファイルを、ソース構造内に存在していたのと同じフォルダーに復元できます。すべて同じフォルダとサブフォルダが存在する必要があります-フォルダは作成されません。

  7. [復元]をクリックすると、復元ダッシュボードに戻り、復元操作の進行状況を確認できます。

Restoreインスタンスは、一定期間非アクティブになるとシャットダウンされます。
